## PostgreSQL

- **[SQL-callable DDL reconstruction keeps landing](https://github.com/postgres/postgres/commit/2af70e9)** — the `pg_get_*_ddl()` family (in-core replacement for "shell out to `pg_dump` to get a CREATE statement") got two fix-up commits this week: Tom Lane [corrected the variadic declarations](https://github.com/postgres/postgres/commit/2af70e9), Andrew Dunstan [switched the option arguments to named booleans](https://github.com/postgres/postgres/commit/d6ed87d); the companion hackers thread "[PATCH] Add pg_get_table_ddl()" drew ~10 messages. `[committed]`
- **[Inherited CHECK constraints can no longer be silently weakened](https://github.com/postgres/postgres/commit/0cd17fd)** — a child table could relax an inherited CHECK, breaking the parent's advertised invariant across the hierarchy; fixed Jun 27 with two follow-up doc commits clarifying `ALTER CONSTRAINT` and inherited-constraint behavior. *(Andrew Dunstan)* `[committed]`
- **[COPY TO (FORMAT json) now respects the column list's order](https://github.com/postgres/postgres/commit/effb923)** — `COPY t (b, a) TO … (FORMAT json)` emits keys in the requested order, matching text/CSV semantics. *(Andrew Dunstan)* `[committed]`
- **[Two beta fixes for temporal `FOR PORTION OF`](https://github.com/postgres/postgres/commit/a272a58)** — the volatility check on SQL:2011 period bounds [moved into the planner](https://github.com/postgres/postgres/commit/a272a58), and [foreign-table child partitions are now rejected](https://github.com/postgres/postgres/commit/a40fdf6) under `FOR PORTION OF`. *(Peter Eisentraut)* `[committed]`
- **[`MERGE`/`SPLIT PARTITION` honors `default_tablespace`](https://github.com/postgres/postgres/commit/cdae794)** — partition reshaping no longer quietly places the result on the wrong tablespace. *(Alexander Korotkov)* `[committed]`
- **[UUID week](https://github.com/postgres/postgres/commit/6468f7a)** — the `uuid`→`bytea` cast is now [leakproof](https://github.com/postgres/postgres/commit/6468f7a) (quals using it can be pushed below RLS/security-barrier views), while the CommitFest collected SIMD-accelerated UUID parsing, `MIN`/`MAX` aggregates for `uuid`, `UUIDv6` support in `uuid_extract_timestamp()`, and a bug report for `uuidv7` accepting pre-1970 dates. *(Masahiko Sawada + CF)* `[committed + patches]`
- **[`pg_stat_io` stops flagging autovacuum-launcher extends as illegal](https://github.com/postgres/postgres/commit/4abf411)** — the autovacuum launcher scans catalog tables, and on-access pruning can now trigger visibility-map extends, tripping an assertion in pg_stat_io; the fix removes the launcher from the assertion list. *(Melanie Plageman)* `[committed]`
- **Beta-hardening grab-bag** — [out-of-bounds read in the autoprewarm worker](https://github.com/postgres/postgres/commit/dac3660) *(Tomas Vondra)*; [correctness pass over the online data-checksums worker](https://github.com/postgres/postgres/commit/a4f02ca) *(Heikki Linnakangas)*; [unsafe order of operations in `ResourceOwnerReleaseAll()`](https://github.com/postgres/postgres/commit/ef01ca6) and a [null-pointer crash in the ECPG compiler](https://github.com/postgres/postgres/commit/7f5e0b2) *(Tom Lane)*; [ModifyTable FDW arrays re-indexed when pruning result relations](https://github.com/postgres/postgres/commit/b43f8aa) — the BUG #19484 segfault *(Amit Langote)*; [`\crosstabview` honors `\pset` boolean/null display](https://github.com/postgres/postgres/commit/4df5fe3) *(Álvaro Herrera)*; [collation-lookup failure avoided for `"char"` columns](https://github.com/postgres/postgres/commit/b574fec) *(Tom Lane)*. `[committed]`

## Research & cutting edge

- **[DiStash: A Disaggregated Multi-Stash Transactional Key-Value Store](https://arxiv.org/abs/2606.27979)** — a transactional KV design for disaggregated hardware that manages key-value pairs across heterogeneous storage pools (DRAM, SSD, HDD, NVM) within a single transaction, with stashes configurable as ephemeral or durable and data inclusive or tiered. _[paper]_
- **[BtrLog: Low-Latency Logging for Cloud Database Systems](https://arxiv.org/abs/2606.27051)** — a reusable cloud logging service that replicates log records across SSD-backed nodes in a single network round trip for synchronous durability, then archives to object storage asynchronously off the latency-critical write path to keep costs low. _[paper]_
- **[CV-Rules: Serializability Verification of Concurrency Control Protocols](https://arxiv.org/abs/2606.25409)** — machine-checkable serializability arguments for CC protocols, using formal methods to prove isolation guarantees hold under concurrency. _[paper · missed by the original edition]_
- **[Query Cost Model Calibration in Confidential Virtual Machines](https://arxiv.org/abs/2606.26385)** — planner cost constants drift inside TEEs/CVMs; directly relevant to anyone running Postgres in confidential compute. _[paper · missed by the original edition]_

# DBMS Weekly — 2026-06-29 (Jun 29 – Jul 5)

## 数据库内核与发布

- **[MySQL 26.7 Community Server Early Access](https://blogs.oracle.com/mysql/mysql-community-server-26-7-early-access-release)** — 社区版首次获得 Thread Pool 插件（此前仅企业版）；新增并行复制 APPLIER_VERSION=2 模式，改进 backlog 恢复；InnoDB undo 日志截断不再依赖本地临时文件，崩溃恢复更可靠；支持 OpenSSL 3.5+ 的混合/纯 ML-KEM 后量子密钥交换。 *(Oracle MySQL)*
- **[ClickHouse 26.6 发布](https://clickhouse.com/blog/clickhouse-release-26-06)** — 本季度大版本：56 项新特性、79 项性能优化。亮点包括 Hypothetical Skip Index（`EXPLAIN WHATIF` 评估索引效果）、Cascading Refreshable Materialized View（`REFRESH DEPENDS ON`）、实验性 Continuous/Streaming Query（`SELECT ... STREAM`）、MPP 风格多阶段分布式执行（scatter/gather/broadcast/shuffle）、Mapbox Vector Tiles (MVT) 地理空间 SQL 生成。 *(ClickHouse)*
- **[Postgres 19 Beta 1 功能概览](https://www.snowflake.com/en/blog/engineering/postgresql-19-features-beta/)** — Snowflake 工程团队梳理 PG19 Beta 1 核心特性：SQL/PGQ 图查询、`REPACK TABLE CONCURRENTLY` 在线表重组、并行 autovacuum、在线 checksum 开关、JIT 默认关闭、LZ4 成为默认 TOAST 压缩算法、`GROUP BY ALL` 语法糖。 *(Snowflake Engineering)*

## 社区热议

- **[Looking Ahead to Postgres 19](https://www.snowflake.com/en/blog/engineering/postgresql-19-features-beta/)** — Snowflake 的 PG19 预览引发了跨引擎大讨论：轻量级连接 vs 高内存占用、物化视图（PG 的 REFRESH vs SQL Server Indexed View）、MariaDB/MySQL 的历史债务（无事务 DDL、无 MERGE、utf8mb3 遗留）、Oracle 的 `"" = NULL` 行为、PG 列存扩展生态（Citus、Hydra、pg_mooncake、deltax）。 *(Hacker News · 218 分, 19 条)*
- **[Postgres data stored in Parquet on S3: LTAP architecture explained](https://www.databricks.com/blog/lakebase-ltap-rethinking-database-storage)** — Databricks 详解 Lakebase 的 LTAP（Lake Transactional/Analytical Processing）架构：基于 serverless Postgres，数据以 Parquet 格式直接存储在 S3 上，OLTP 和 OLAP 共享同一份数据，无需 CDC 或镜像。讨论集中在存储格式选择、写放大问题、以及与传统 PG 生态的兼容性。 *(Hacker News · 176 分)*
- **[Postgres transactions are a distributed systems superpower](https://times.hntrends.net/story/48765639)** — 一篇引发热议的文章：将应用数据和 Workflow 状态放在同一个 Postgres 数据库中，利用单事务更新消除分布式工作流中的部分失败和幂等性问题。 *(Hacker News · Jul 3)*

## 分布式数据与系统工程

- **[Experience Graphs: The Data Foundation for Self-Improving Agents](https://arxiv.org/abs/2606.29823)** — Daniel Abadi（Meta）等提出 Trellis 架构，将 AI Agent 的搜索过程建模为数据库访问模式：候选方案选择是 frontier query、相似历史会话检索是向量图遍历、训练数据提取是物化视图。类比日志如何从临时输出演变为可查询的持久状态，Experience Graph 将 Agent 推理从一次性计算变成可累积的制度资产。Meta 内部 GPU Kernel 优化器的生产数据表明跨会话复用可将目标加速比提升约 10×、token 成本降低 52%。 *(Abadi et al. · Meta)* _[paper]_
- **[CADENZA: Intent-Dependent Plan Spaces for Semantic Query Processing](https://arxiv.org/abs/2606.29151)** — 将自然语言意图编译为任务特定算子 DAG。不同于传统的单一执行计划空间，CADENZA 根据语义意图动态构建差异化的计划搜索空间——"打破单体优化器"的思路。同时被 VLDB 2026 和 SIGMOD 2027 接收。 *(Ha, Park, Han)* _[paper]_

## 商业引擎（SQL Server, Oracle, MySQL, …）

- **[Oracle 引入 DML NOWAIT 子句](https://blogs.oracle.com/ace/monthly-product-pulse-july-2026-oracle-technical-resources)** — `INSERT`、`UPDATE`、`DELETE`、`MERGE` 均支持 `NOWAIT`，遇到行锁时立即返回错误而非阻塞等待。Oracle 7 月技术月报还涵盖外键数据完整性深入分析、SQL 性能调优之逻辑 I/O 详解、Autonomous AI Database 的 AWS KMS 跨区域 Data Guard 正式 GA。 *(Oracle ACE)*
- **[SAP HANA Cloud Q2 2026: 数据库内嵌 AI 函数](https://community.sap.com/t5/technology-blog-posts-by-sap/what-s-new-in-sap-hana-cloud-july-2026/ba-p/14419199)** — 新增 `AI_TABULAR_PREDICTION` 和 `AI_TEXT_COMPLETION` SQL 函数，直接在数据库引擎内调用 LLM；基于数据库元数据（表、视图、列、关系）自动生成知识图谱以驱动 AI 数据发现；Performance Class Advisor 每周自动给出实例规格建议。 *(SAP)*

## 学术前沿

- **[Large Databases Need Small, Open-Weight Language Models](https://arxiv.org/abs/2606.31808)** — 量化后的本地开源小模型（16GB VRAM）在数据库关系算子增强任务上达到与闭源 API 同等的准确率，但成本降低 390×、延迟降低 3.8×。对 Text-to-SQL 和 NL2SQL 落地有直接指导意义。 *(Glenn, Samuel)* _[paper]_
- **[HNSW with Accuracy Guarantees Using Graph Spanners](https://arxiv.org/abs/2607.02338)** — 提出 "Certify-then-Rectify" 框架：先对 HNSW 搜索结果做统计认证，再通过图 Spanner 理论在最坏情况下精确恢复漏召回的点。为标准 HNSW 首次提供精度保证机制。已投稿 VLDB 2027。 *(Li, Mittal, Rana et al.)* _[paper]_
- **[SemJoin: Semantic Join Optimization](https://arxiv.org/abs/2606.29532)** — 基于 LLM Agent 的管道，在 Cluster Join 和 Classifier 两种策略间动态路由语义 Join，比 ABJ 方法 F1 提升 20–33 分且 token 成本更低。VLDB 2026 NOVAS Workshop。 *(Gou, Banerjee, Wang, Liu)* _[paper]_
- **[TVA: A Version-aware Temporal Graph Storage System for Real-time Analytics](https://arxiv.org/abs/2607.00406)** — 面向实时分析的版本感知时序图存储系统，高效管理图数据的多版本快照。已接收于 VLDB 2026。 *(Li, Zhao, Dong et al.)* _[paper]_
- **[CLIP: Lightweight Cosine-Law-Based Inverted-List Pruning for IVF](https://arxiv.org/abs/2606.29968)** — 利用余弦相似度的几何性质对 IVF 倒排列表做轻量剪枝，在不损失召回率的前提下加速向量检索。 *(Song, Lu, Zhou et al.)* _[paper]_
- **[DiStash: A Disaggregated Multi-Stash Transactional Key-Value Store](https://arxiv.org/abs/2606.27979)** — 解耦的多 Stash 事务 KV 存储，将存储层与计算层完全分离。扩展版发表于 TPC Technology Conference (VLDB 2025)。 *(Gao, Nguyen, Li, Ghandeharizadeh)* _[paper]_
